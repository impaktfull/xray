# Impaktfull Inspector

A comprehensive Flutter debugging toolkit that captures and displays network calls, logs, and key-value storage — all inspectable in real-time via a remote UI.

## What's included

| Package | Description |
|---|---|
| `impaktfull_inspector` | Core package — network, log, and key-value inspectors + HTTP server |
| `impaktfull_network_dio_interceptor` | Dio HTTP interceptor |
| `impaktfull_network_serverpod_interceptor` | Serverpod HTTP client interceptor |
| `impaktfull_shared_preferences_inspector` | SharedPreferences key-value inspector |
| `impaktfull_secure_storage_inspector` | flutter_secure_storage inspector |
| `impaktfull_inspector_ui` | Flutter UI that connects to a running inspector server |
| `impaktfull_inspector_core` | Pure Dart models and protocol constants (no Flutter dependency) |

## Architecture overview

```
Your Flutter app
       │
       ├── ImpaktfullNetworkInspector   ◄── ImpaktfullNetworkDioInterceptor
       ├── ImpaktfullLogInspector
       ├── ImpaktfullSharedPreferencesInspector
       └── ImpaktfullSecureStorageInspector
                        │
              ImpaktfullInspectorServer  (HTTP + WebSocket + UDP discovery)
                        │
              ImpaktfullInspectorUiController  (separate app or debug panel)
                        │
                Inspector UI screens
```

The inspector server runs inside your app and broadcasts data over HTTP/WebSocket. A separate UI (another device, simulator, or debug panel) connects to it via `ImpaktfullInspectorUiController`.

## Design principles

- **No singletons** — all dependencies are passed explicitly via constructors.
- **ValueNotifier reactivity** — all data exposed as `ValueNotifier` for Flutter rebuilds.
- **No disk I/O** — all captured data lives in memory only.
- **Composable** — add only the inspectors you need.
